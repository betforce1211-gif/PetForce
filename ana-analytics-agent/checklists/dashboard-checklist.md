## Checklist Items

### Chart Selection & Design
- [ ] Right chart type chosen for the data (line, bar, pie, scatter, table, KPI)
- [ ] Chart serves a clear decision-making purpose
- [ ] Chart junk removed (no 3D effects, appropriate to data)
- [ ] Visual hierarchy supports scannability (KPIs first)

### Color & Accessibility
- [ ] Consistent, accessible color palette used
- [ ] Color contrast ratios meet standards (4.5:1 minimum)
- [ ] Color not used as only differentiator (patterns/labels included)
- [ ] Colorblind-friendly palette tested
- [ ] Red/green not sole indicators

### Context & Clarity
- [ ] Context provided (comparisons, goals, trends included)
- [ ] Tooltips added for detail on demand
- [ ] Chart titles descriptive and clear
- [ ] Axes labeled appropriately
- [ ] Y-axis not truncated to mislead

### Responsive Design
- [ ] Mobile-first design approach used
- [ ] Dashboard scales appropriately to desktop
- [ ] Layout tested on actual devices
- [ ] Touch targets adequate on mobile

### States & Performance
- [ ] Loading states designed and implemented
- [ ] Error states handled gracefully with retry options
- [ ] Data limits respected (appropriate aggregation, pagination)
- [ ] Performance optimized (caching, lazy loading)
