### Metrics

#### KPI Cards (Top Row)

**Active Households (This Week)**

- **Value:** Count of households with activity in last 7 days
- **Comparison:** vs. last week (% change)
- **Color:** Green if up, red if down
- **Goal:** 500 active households

**Daily Active Users (Today)**

- **Value:** Unique users today
- **Comparison:** vs. last week same day
- **Color:** Green if up, red if down
- **Goal:** 1,000 DAU

**Day 7 Retention**

- **Value:** % of users who return on day 7
- **Comparison:** vs. previous cohort
- **Color:** Green if > 40%, yellow 35-40%, red < 35%
- **Goal:** 40%

#### Charts

**Daily Active Users Trend**

- **Type:** Line chart
- **X-axis:** Date (last 30 days)
- **Y-axis:** User count
- **Data:** One data point per day
- **Interaction:** Hover to see exact count

**New Registrations (Weekly)**

- **Type:** Vertical bar chart
- **X-axis:** Day of week
- **Y-axis:** Registration count
- **Data:** Last 7 days
- **Insight:** Identify high-traffic days

**Household Growth (Weekly)**

- **Type:** Vertical bar chart
- **X-axis:** Day of week
- **Y-axis:** New households
- **Data:** Last 7 days
- **Pattern:** Should correlate with registrations

**Feature Adoption**

- **Type:** Horizontal bar chart
- **X-axis:** Percentage (0-100%)
- **Y-axis:** Feature names
- **Data:** % of active users who used each feature
- **Sorted:** By adoption rate (descending)

- Step-by-step conversion rates

- Distribution: Email vs OAuth

- Histogram of verification times
- Identifies email deliverability issues
- Target: Most users verify in < 5 minutes

- Week-over-week cohort comparison
- Shows impact of product changes
- Annotated with deployment dates

### Metrics

**DAU/MAU Ratio (Stickiness)**

- Daily/Monthly active users
- Industry benchmark: 30-40%
- Higher = more engaged users

**Retention Cohorts**

- Table showing Day 1, 7, 30 retention
- One row per weekly cohort
- Identify retention improvements

**User Activity Distribution**

- Segments: Daily, Weekly, Monthly, Inactive
- Pie chart or stacked bar
- Goal: > 25% daily active

**At-Risk Users**

- Users with declining engagement
- Prioritize for customer success outreach
- Potential churn prevention

## Dashboard Best Practices

### Design Principles

**1. Most Important First**

- Top of page = most critical metric
- F-pattern reading (left to right, top to bottom)

**2. Comparison Context**

- Always show: vs. last week/month
- Color code: Green (up), Red (down)
- Provide targets/goals

**3. Actionable Insights**

- Don't just show data
- Add annotations ("Spike due to deploy")
- Suggest next steps

**4. Drill-Down Capability**

- Click chart â†’ see detail table
- Click data point â†’ see user list
- Filters persistent in URL

**5. Accessible Design**

- WCAG 2.1 AA compliant
- Color-blind safe palettes
- Keyboard navigation
- Screen reader friendly

### Performance

**Loading Time Targets:**

- Initial load: < 2 seconds
- Filter change: < 500ms
- Chart update: < 200ms

**Optimization Techniques:**

- Server-side aggregation
- Client-side caching
- Lazy load charts below fold
- Debounce filter changes
